# Reproduction Infrastructure

This directory contains the infrastructure for running and tracking
reproducible results from the Life-Cycle-Prime-Time project.

## Quick Start

From the repository root:

```bash
# Full reproduction (all 3 agent models)
bash reproduce.sh

# Quick validation (tests + single estimation)
bash reproduce_min.sh
```

Both scripts automatically:
- Log all output to `reproduce/logs/`
- On success, write a benchmark JSON to `reproduce/benchmarks/results/autogenerated/`
  capturing system characteristics, timing, package versions, and git state

## How It Works

### Logging

Every run of `reproduce.sh` or `reproduce_min.sh` creates a timestamped
log file in `reproduce/logs/`. A `latest.log` symlink always points to
the most recent log. Log files are gitignored.

### Benchmarking

When a reproduction run succeeds (exit code 0), a benchmark JSON file is
automatically written to `reproduce/benchmarks/results/autogenerated/`.
The file captures:

- **Timing**: start/end timestamps, wall-clock duration
- **System**: OS, CPU model/cores, memory, disk type
- **Environment**: Python version, key package versions, venv type
- **Git**: commit hash, branch, dirty flag
- **Metadata**: user, session ID, CI flag

Benchmark files are named: `{kind}_{YYYYMMDD-HHMM}_{duration}s.json`

Failed runs do **not** generate benchmarks (only successes are recorded).

### Viewing Results

```bash
# Formatted table of all benchmark results
bash reproduce/benchmarks/benchmark_results.sh

# Filter by kind
bash reproduce/benchmarks/benchmark_results.sh min
bash reproduce/benchmarks/benchmark_results.sh full

# View the latest result in detail
jq . reproduce/benchmarks/results/latest.json
```

### Saving Reference Results

To preserve a benchmark as a reference (tracked in git):

```bash
cp reproduce/benchmarks/results/autogenerated/<file>.json \
   reproduce/benchmarks/results/saved/
git add reproduce/benchmarks/results/saved/<file>.json
```

### Checking Dependencies

```bash
bash reproduce/check_dependencies.sh
```

This verifies that `python3` (3.12+), `uv`, and `git` are available.

### Disabling Benchmarks

Set `BENCHMARK=false` before running:

```bash
BENCHMARK=false bash reproduce.sh
```

## Shared Utilities (`reproduce_utils.sh`)

Both `reproduce.sh` and `reproduce_min.sh` source this file, which
provides:

| Function | Purpose |
|---|---|
| `log LEVEL message` | Print and optionally log a message |
| `init_logging KIND` | Start a new log file for this run |
| `benchmark_start KIND` | Begin timing a benchmark |
| `benchmark_end EXIT_CODE` | Write benchmark JSON on success |
| `print_summary EXIT_CODE` | Print a final summary |
| Error handler (ERR trap) | Log failures with line numbers |
| Exit handler (EXIT trap) | Auto-print summary and record benchmark |
