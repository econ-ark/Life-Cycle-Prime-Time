# Benchmarking System

Benchmark results are automatically generated when `reproduce.sh` or
`reproduce_min.sh` complete successfully.

## Result Format

Each benchmark is a JSON file following `schema.json`. Key fields:

| Field | Description |
|---|---|
| `reproduction_kind` | `"full"` or `"min"` |
| `duration_seconds` | Wall-clock time |
| `exit_status` | 0 = success |
| `system.cpu` | CPU model, cores, architecture |
| `system.memory` | Total/available RAM in GB |
| `environment.python_version` | Python version |
| `environment.key_packages` | Versions of econ-ark, HARK, estimagic, etc. |
| `git.commit` | Git commit hash |

## Directory Structure

```
results/
├── autogenerated/   # Written automatically (gitignored)
│   └── .gitignore
├── saved/           # Copy here to track in git
│   └── .gitkeep
└── latest.json      # Symlink to most recent result
```

## Usage

```bash
# View all results
bash benchmark_results.sh

# View latest in detail
jq . results/latest.json

# Compare durations across runs
jq '.duration_seconds' results/autogenerated/*.json

# Average duration
jq -s 'map(.duration_seconds) | add/length' results/autogenerated/*.json

# Save a reference result
cp results/autogenerated/<file>.json results/saved/
```

## Filename Convention

```
{kind}_{YYYYMMDD-HHMM}_{duration_padded}s.json
```

Examples:
- `full_20260310-1430_000847s.json` -- full run, ~14 minutes
- `min_20260310-1502_000092s.json` -- min run, ~92 seconds
